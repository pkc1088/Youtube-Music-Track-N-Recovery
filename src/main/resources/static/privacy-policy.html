Privacy Policy

Last Updated: July 17, 2025

This service uses Google’s YouTube API Services (YouTube Data API v3). By using this service, you understand and agree that your Google user data (such as playlists and video metadata) may be processed through the API. This is subject to the YouTube Terms of Service (https://www.youtube.com/t/terms) and Google Privacy Policy (https://policies.google.com/privacy).

1. Information Collected

We collect the following information through Google OAuth2 authentication:

  • Basic Google account information (email address, unique identifier)
  • Access token and refresh token for accessing YouTube data
  • User’s YouTube playlists and video metadata

2. Purpose of Use

This service uses the sensitive scope `youtube.force-ssl` granted by Google to implement playlist item deletion and addition. This scope is explicitly consented to by users through Google OAuth authentication. No other scopes are requested. The collected information is used only for the following purposes:

  • User identification for providing the service
  • Tracking changes to YouTube playlists
  • Deleting broken videos and adding new ones to playlists

3. Third-Party Sharing

This service transmits Google user data (such as playlists and tokens) to Google APIs (YouTube Data API v3) to perform playlist modification as requested by users. User data is not shared with any other third parties (e.g., advertisers, external servers) except in the following cases:

  • For normal service operation with Google YouTube API
  • When required by law

4. Use of Cookies and Similar Technologies

This service uses web browser cookies (JSESSIONID) and sessions to maintain user authentication status. These cookies are essential for user login processing and securing the OAuth2 authentication flow. They are not used for any other purposes (e.g., advertising).

5. Data Storage and Deletion

User data is safely managed according to the following standards:

  • Detect broken videos and remove them from the playlist
  • Use information of broken videos to find replacements on YouTube
  • Add replacement videos to the corresponding playlist

6. Data Deletion and Revocation of Access

User data is safely managed according to the following standards:

  • By clicking the ‘Delete Account’ button after logging in, all personal and linked data (registered accounts, playlists, recovery history) stored by this service will be immediately deleted.
  • When a playlist registration checkbox is unchecked, that playlist is immediately removed from FixMyPlaylist’s managed list. This does not affect the actual YouTube playlist.
  • Collected data is stored in Google Cloud SQL and protected through encryption and access controls.
  • Refresh tokens can be invalidated via the Google OAuth revocation API, and users can directly revoke OAuth access at any time via the Google Security Settings page (https://security.google.com/settings/security/permissions).
  • This service does not store or access user data beyond the purpose of providing the service. If requested by the user, data will be deleted at any time.

7. Security Measures

To protect data, we implement the following measures:

  • Strict access control based on the principle of least privilege
  • Regular security audits

8. Summary of Google User Data Processing Flow

The overall service registration and user data processing follow this sequence:

  • The user grants access to their email and YouTube data via OAuth authentication.
  • This service analyzes the information to detect broken videos in the playlist.
  • Once per day, the detected items are used to infer song information and insert replacement videos via the YouTube API.
  • In this process, tokens and playlist information are stored in Google Cloud SQL.
  • When the user deletes their account, all information is immediately deleted, and the Google OAuth token is also revoked.
